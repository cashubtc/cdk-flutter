# AGENTS.md

> Guidance for AI agents and automated workflows contributing to **cdk_flutter**.
> For human contributors, the [README](README.md) and the parent [CDK](https://github.com/cashubtc/cdk) repo are the canonical references.

## Overview

`cdk_flutter` is a Flutter plugin that exposes the [Cashu Development Kit (CDK)](https://github.com/cashubtc/cdk) Rust crate to Dart via [`flutter_rust_bridge`](https://cjycode.com/flutter_rust_bridge/) `2.11.1` (pinned). It builds cross-platform Cashu wallets with native performance on Android, iOS, macOS, Linux, and Windows.

The Rust side lives under `rust/`. The Dart side lives under `lib/`. Bindings between them are **code-generated** — see [Code Generation Boundaries](#code-generation-boundaries) below.

## Common Commands

All common commands are defined in the [`justfile`](justfile). Run `just` with no arguments to list them.

| Task | Command |
|---|---|
| Install deps and build Rust | `just setup` |
| Regenerate bindings | `just generate` |
| Build Rust library only | `just build` |
| Run all checks (analyze + cargo check + clippy) | `just check` |
| Format (dart + rust) | `just format` |
| Clean all build artifacts | `just clean` |
| Run example app | `just run device=<device>` (e.g. `macos`, `linux`, `ios`) |
| Run integration tests | `just test device=<device>` |

Regenerate bindings any time you change a function signature, struct, or enum under `rust/src/api/`.

## Environment Setup

- **Flutter** `>= 3.3.0` with Dart SDK `^3.6.0` (see [`pubspec.yaml`](pubspec.yaml))
- **Rust** stable toolchain (install via [rustup](https://rustup.rs))
- **`flutter_rust_bridge_codegen`** `2.11.1` — install with:
  ```bash
  cargo install flutter_rust_bridge_codegen --version 2.11.1 --locked
  ```
  The codegen version **must match** the `flutter_rust_bridge` runtime version in `pubspec.yaml` and `rust/Cargo.toml`.
- **`just`** (optional but recommended) — install via `brew install just`, `cargo install just`, or [`casey/just`](https://github.com/casey/just).
- **Nix** (optional) — a `flake.nix` is provided for reproducible dev shells: `nix develop`.

Platform-specific tooling (Xcode for iOS/macOS, Android SDK + NDK for Android, etc.) is required for each target platform you build against.

## Repo Conventions

### Commit messages

Use [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add new feature
fix: correct a bug
docs: update documentation
chore: maintenance / tooling
```

Optional scopes reflect the boundary: `feat(wallet):`, `fix(bindings):`, `chore(rust):`, `docs(example):`.

### Pull requests

- Keep PRs small and focused on a single concern.
- Always run `just check` and `just format` before pushing.
- If your change touches `rust/src/api/`, regenerate bindings with `just generate` and commit the regenerated `lib/src/rust/` alongside your Rust change in the **same PR**. Do not leave generated code out of sync with hand-written code.
- Reference related issues in the PR description (e.g. `Closes #123`).

### Testing

- Unit tests live next to the code they test under `rust/src/` (run via `cargo test`).
- Integration tests for the Dart API live in [`example/integration_test/`](example/integration_test/) (run via `just test`).
- The example app targets the public `testnut` mint (`https://testnut.cashu.space`) by default. Do not hardcode other mints in tests.

## Code Generation Boundaries

This repository mixes **hand-written** and **generated** source. Agents must understand the boundary before editing.

### Hand-written (edit freely)

- `rust/src/api/*.rs` — the public Rust API surface exposed to Dart. Each file maps to a domain:
  - `wallet.rs`, `mint.rs`, `token.rs`, `payment_request.rs`, `bolt11.rs`, `bitcoin.rs`, `key.rs`, `init.rs`, `error.rs`
- `rust/src/lib.rs` — crate root.
- `lib/cdk_flutter.dart` — public Dart entrypoint (re-exports).
- `lib/widgets.dart` — Flutter widgets (`WalletBalanceBuilder`, `MintQuoteBuilder`, `SendBuilder`, `ReceiveBuilder`, `MeltQuoteBuilder`, `TransactionListBuilder`, `WalletProvider`, etc.) built on top of the generated bindings.
- `example/` — the example app (Flutter source, freely editable).

### Generated (do NOT hand-edit)

- `rust/src/frb_generated.rs` — FFI glue generated by `flutter_rust_bridge_codegen`.
- `lib/src/rust/` — the Dart side of the bindings (everything under this directory, including `frb_generated.dart`, `frb_generated.io.dart`, `frb_generated.web.dart`, `lib.dart`, and all files under `api/`). Every file in this tree starts with `// This file is automatically generated, so please do not edit it.`

If you need to change something in the generated tree, change the corresponding hand-written Rust source under `rust/src/api/` and run `just generate`.

### Vendored tooling

- `cargokit/` — vendored Cargo-Flutter build integration. Do **not** modify unless upgrading cargokit intentionally.

## Autonomous vs. Human-Required Changes

### Safe for agents to do autonomously

- Documentation (README, AGENTS.md, doc comments, example app docs)
- Formatting and lint fixes (`just format`, addressing `just check` warnings)
- Adding or improving tests (unit, integration, example-app scenarios)
- Non-breaking additions to the example app
- Dependency patch bumps (e.g. `2.11.1` → `2.11.2`) when the upstream changelog confirms no API break

### Requires human review and decision

- **Breaking changes to `rust/src/api/`** — signatures here shape the generated Dart API. Any rename, parameter change, or return-type change is a public API break for every consumer.
- **`cdk` / `cdk-common` / `cdk-sqlite` version bumps** — these are protocol-level crates; a minor version bump can shift semantics. Coordinate with [thesimplekid](https://github.com/thesimplekid) and track the CDK [`CHANGELOG`](https://github.com/cashubtc/cdk/blob/main/CHANGELOG.md).
- **`flutter_rust_bridge` or `flutter_rust_bridge_codegen` version bumps** — the runtime and codegen versions must stay in lockstep across `pubspec.yaml`, `rust/Cargo.toml`, and the tool binary.
- **Cryptographic or signing code** — anything touching keys, BIP39 mnemonics, blind signatures, P2PK, or DLEQ proofs. Defer to the Rust-side review in the parent [`cdk`](https://github.com/cashubtc/cdk) repo.
- **Platform-native glue** — iOS entitlements, Android NDK version pinning, macOS signing, Windows/Linux CMake — these frequently surface device-specific bugs that require human testing.
- **Database migrations** — `wallet.sqlite` stores user funds. Schema changes must preserve existing user data.
- **Cashu protocol (NUT) behavior** — protocol-level behavior is defined in [`cashubtc/nuts`](https://github.com/cashubtc/nuts). Implementation changes that affect protocol conformance must reference the relevant NUT.

When in doubt, open an issue or discussion before submitting a large unsolicited change.

## Architecture Overview

```
cdk-flutter/
├── rust/                       # Rust crate — the bridge source of truth
│   ├── src/
│   │   ├── api/                # Hand-written — public API exposed to Dart
│   │   │   ├── wallet.rs       #   Wallet, MultiMintWallet, balance, send, receive
│   │   │   ├── mint.rs         #   Mint quote (invoice → ecash)
│   │   │   ├── token.rs        #   Token encoding/decoding
│   │   │   ├── payment_request.rs
│   │   │   ├── bolt11.rs       #   Lightning invoice parsing
│   │   │   ├── bitcoin.rs      #   BIP39, address helpers
│   │   │   ├── key.rs          #   Keys and signing
│   │   │   ├── init.rs         #   Library init
│   │   │   └── error.rs        #   Error types
│   │   ├── frb_generated.rs    # Generated FFI glue
│   │   └── lib.rs              # Crate root
│   └── Cargo.toml              # Pins cdk 0.16.x, flutter_rust_bridge 2.11.1
│
├── lib/                        # Dart package — public Flutter API
│   ├── cdk_flutter.dart        # Public entrypoint (re-exports)
│   ├── widgets.dart            # Flutter Builder widgets
│   └── src/rust/               # Generated Dart bindings (do not edit)
│
├── example/                    # Reference example app
│   ├── lib/                    # home, mint, send, receive, melt, request screens
│   └── integration_test/       # Dart integration tests
│
├── android/ ios/ macos/
│   linux/ windows/             # Platform plugin scaffolding (ffiPlugin: true)
│
├── cargokit/                   # Vendored Cargo + Flutter build glue
├── flake.nix                   # Nix dev shell
├── flutter_rust_bridge.yaml    # Codegen config (rust/src/api → lib/src/rust)
└── justfile                    # Canonical command reference
```
